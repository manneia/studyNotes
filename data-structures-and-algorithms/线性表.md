# 第二章 线性表

基本实现方式: 顺序实现和链式实现
链式实现不支持随机存取

## 定义
线性表是一种线性结构。在这种结构中存在唯一的"第一个"元素,唯一的"第二个"元素.以此类推.线性表中各个元素依次排列

是同类型数据元素构成的有限序列

1. 所有数据元素都必须是相同类型的
2. 数据元素的次序就是他们在表中的排列次序
3. 第一个元素是 $$a_0$$,称为表头或开始元素
4. 第n个也是最后一个元素是 $$a_{n-1}$$,称为表尾或终端元素
5. 元素的个数n称为表长
6. n=0时称为空表

### 顺序存储

抽象数据类型定义:

```
ADT LinerList {

	数据部分:
		L
	操作部分:
		int initList(LinearList *L) // 初始化线性表
		int clear(LinearList *L) // 置空表
		int isEmpty(LinearList *L) // 判断表是否为空
		int isFull(LinearList *L) // 判断表是否已满
		int length(LinearList *L) // 返回表的长度
		int insertList(LinearList *L, Position Pos, ElemType x) // 在pos位置处插入元素x
		int removeList(LinearList *L, Position Pos, ElemType x) // 删除pos处的元素,并通过x返回
		int setValue(LinearList *L, Position Pos, ElemType x) // 给pos位置处的元素赋值为x
		int getValue(LinearList *L, Position Pos, ElemType x) // 获取位置pos处的元素
		Position find(LinearList *L,  ElemType x) // 返回元素x第一次出现的位置
}
```

基本思想: 一组连续的存储单元依次存储各个元素,将线性表中的个数据元素,按照其逻辑次序,依次保存
线性表中逻辑上相邻的两个元素,保存在数组相邻的两个单元中
数组是内存中一片连续的空间,相邻的两个单元在内存中的实际地址也是相邻的.表明线性表中,逻辑上相邻,其存储地址也相邻

### 链式存储 
- 线性表的各个元素保存在各自的存储空间中,形成一个个的结点
- 这些结点在内存中的地址不要求是相邻的,他们之间通过指针链接起来,这种方式保存的线性表称为链表

1. 链式存储是一种动态灵活的存储方式,不要求预先分配一块连续的存储空间,而是按需分配
2. 不要求分配的空间必须是相邻的,而是由系统决定分配的具体位置,既可以相邻也可以不相邻
3. 执行插入以及删除操作时,不在需要移动元素以保证存储空间的相邻性

#### 单链表

单链表是由一组动态分配的结点形成的链表,每个结点保存线性表中的一个元素以及一个指针,指针指向保存其后记元素的结点
```
int initList(LinkList *head) {

	(*head) = (LinkList)malloc(sizeof(LinkNode));
	if (*head == NULL) {
		return false;
	}
	(*head)->data = 0;
	(*head)->next=NULL;
}
```c++

循环链表: 修改单链表的定义,将表尾结点的指针指回头节点,从而形成循环链表

双向链表: 链表的结点的左指针指向前驱节点,右指针指向后继结点

双向循环链表: 将双向链表的尾结点的后继结点指向表头结点


## 顺序实现和链式的比较

- 顺序表存储每个元素时,空间相对紧凑.并且占用连续的空间
- 数组的每个单元中只需要保存数据本身,没有额外的开销
- 链表的每个结点除了数据元素外,还需要留出单独的空间存放指针.这些保存指针所占用的空间称为结构性开销

n表示当前元素的个数, D表示最多可以在数组中保存的元素个数, P表示指针的存储单元大小 E表示数据元素的存储单元大小
顺序表: D * E
单链表: n * (P + E)
双向链表: 比单链表中每个结点的指针数多一个,所以双向链表的结构性开销是单链表的两倍

顺序表访问操作时, 时间复杂度为O(1)
单链表访问操作的平均和最差时间复杂度为O(n)
插入操作: 单链表为O(1) 顺序表为O(n)
静态链表: 保存在数组中, 兼具顺序表和链表特性,空间不零碎, 插入删除不需要移动元素
